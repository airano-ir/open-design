お客様事例 · Stuart Gardoll

“まっ先に 開くのは Open Design — それ以来、一度も後戻りしていない。”

Stuart Gardoll はクライアント向けに AI を活かしたアプリを世に出しながら、AI でものづくりをするチャンネル Let’s Build を運営している — だから彼は、常に最新のツールとモデルを試し続けている。そんな彼が、まっ先に開くようになったのが Open Design だ。アプリ UI、モーショングラフィックス、プロトタイプを、自分が選んだモデルの上で形にできる、クリエイティブなワークスペース。

背景

あらゆるモデルを試す。必要だったのは、ひとつのクリエイティブな作業場。

Stuart はひとり体制のコンサルティング会社 Connectio を営み、各社が AI を活かしたアプリを世に出すのを支えている — そして、AI でものづくりをする YouTube チャンネル Let’s Build を運営している。(その前は、FlutterFlow のグロースハッカーだった。)最新であり続けることこそが仕事だから、彼は新しいツール、モデル、コーディングエージェントの間を絶えず行き来している。彼が求めているのは、たったひとつの完璧なモデルではない — 足もとのモデルが変わり続けても、変わらずそこにあるクリエイティブな作業場だ。

彼は Claude Design を使っていて、その方向性を気に入っていた。ただ、それは 1 社のモデルに縛られるということでもあった。週単位でモデルを乗り換える人間にとって、そのロックインは形が合わなかった。そんなとき、知り合いが オープンソースの代替ツール について投稿していた — “オープンソースのコミュニティは、できる限りいつも応援したい。だから試してみたんです。”

Claude Design にできることは、すべてできる — しかも 自分が選んだモデルで。それどころか、あちこち縛られていないぶん、もう一歩先まで行ける。
Stuart Gardoll · Connectio
実例 · アプリ制作のワークフロー

世に出たアプリ、その形は Open Design で。

Stuart の Read It — どんな記事でも音声に変えるアプリ — は、いま App Store と Google Play で公開中だ。そのインターフェースが形になる場所が、Open Design。読み上げの流れ、読み込み中の動きの演出、人の手ざわりを感じさせる細やかなタイミングの一つひとつまで。彼はそこで方向性をデザインし、できあがったアーティファクトをコーディングエージェントに渡して、自分の Flutter スタックで作り込んでいく。

“必要なものを何でもここで作って、あとは使い慣れた CLI に引き渡すだけ。流れがすごく気持ちいいんです。”

実例 · コンテンツ制作のワークフロー

彼の動画のモーショングラフィックスは — Open Design 製。

もうひとつのワークフローは、メディアだ。Stuart の Let’s Build 動画の中には、モーショングラフィックスを Open Design と Hyperframes で作っているものがある。自分のブランドカラーで、欲しいものを言葉で伝えるだけ — あとは Open Design が生成し、その場でプレビューし、MP4 に書き出す — そのまま編集に放り込める状態で、しかも視聴者が期待する プロ品質で。

“最新の動画は、モーショングラフィックスをぜんぶ Open Design で作っています。”

そして、それだけにとどまらない

何を作るときも、作業場はひとつ。

リポジトリから、プロトタイプへ

既存の GitHub リポジトリを持ってきて、Open Design で作り直す — 粗削りなツールが、そのまま世に出せる洗練されたものに変わる。

動くアーティファクト

ある朝の実験で、Hacker News を Flipboard 風の新聞に仕立てた — “本当にすごいことになった”ものを、そのままソフトウェアへ落とし込む。

自分だけのデザインシステム

プリセットではなく — 自分のデザインシステムや、繰り返し使えるビジュアルの方向性を試し、それをもとに生成していく。

なぜ、彼の定番になったのか

すでに使っている、すべてとつながる。

Open Design は、Stuart のスタックの残りの部分にすっと収まる。Figma から読み込み、CLI やコーディングエージェントへきれいに引き渡し、さらに日々のプロジェクト管理ツールにもつなげたいと考えている。“このプラットフォームが、ほかのすべてとこれだけつながっているのが気に入っています。”あらゆるツールを試す人間が、あちこち探し回るのをやめた — その理由がこれだ。

Claude Design のような力を — あなた自身のモデルで。

モデルは変わり続ける — でも、あなたのクリエイティブな作業場まで変わる必要はない。Open Design は無料・オープンソースで、あなた自身のマシンと、あなた自身の API キーで動く。使いたいモデルを、そのまま持ち込んでください。
